\subsection{Эволюция нейронных сетей через расширение топологий}

\subsubsection{Постановка задачи и мотивация}

В своей работе \cite{stanley2002neuroevolution} Stanley и Miikkulainen рассматривают нейроэволюцию (NE) как альтернативу классическим методам обучения с подкреплением, где вместо оценки функции ценности (value function) или градиентов политики напрямую реализуется эволюция поведения агента, заданного параметрами нейросети.
Ключевой вопрос: можно ли получить преимущество, эволюционируя топологию сети вместе с весами, по сравнению с традиционными подходами, где оптимизируются только веса при фиксированной архитектуре.

Авторы указывают несколько проблем в классических подходах по эволюции топологии и весов в нейронных сетях (\texttt{TWEANN-подходы}):

\begin{enumerate}

  \item Проблема \textbf{соперничающих соглашений (competing conventions)} — одна и та же функция может быть закодирована разными перестановками скрытых нейронов, что делает кроссовер разрушающим. Кроссовером считаем операцию скрещивания информации из разных генов. Решение этой проблемы будет рассмотрено далее.
  
  \item Сложность \textbf{сохранения инноваций}: новые структурные элементы (нейроны/связи) сначала ухудшают приспособленность, в результате чего агенты быстро вымирают без специальной защиты
  
  \item Отсутствие механизма, который бы \textbf{поощрял минимальные архитектуры} и рост сложности “по мере необходимости”
  
\end{enumerate}


\subsubsection{Основные идеи и архитектура NEAT}

NEAT предлагает три ключевых механизма решения этих проблем:

\begin{enumerate}

  \item \textbf{Исторические метки}

  Каждому новому гену (связи/нейрону) при структурной мутации присваивается уникальный номер инновации. Это позволяет при кроссовере однозначно “выравнивать” гены по происхождению и отличать:

  \begin{itemize}

    \item совпадающие внутри диапазона гены
    \item несоответствующие внутри диапазона
    \item и вне диапазона

    За счёт этого кроссовер между разными топологиями становится осмысленным и не разрушает функциональные подструктуры.

  \end{itemize}

  \item \textbf{Защита при кроссовере}

  Вводится метрика “генетической дистанции” между геномами на основе числа disjoint/excess-генов и различий в весах. Популяция разбивается на виды (species), и внутри вида применяется распределение приспособленности (fitness sharing). Это:

  \begin{itemize}

    \item ограничивает размер каждого вида
    \item не даёт одному “удачному” шаблону подавлять остальные
    \item создаёт “защищённые ниши” для особей с новыми структурами, которым требуется несколько поколений, чтобы стать полезными

  \end{itemize}

  \item \textbf{Инкрементальный рост из минимальной архитектуры}

  Вместо старта с большого случайного множества топологий NEAT начинает с \textbf{минимальных сетей без скрытых нейронов} (только входы–выходы) и постепенно добавляет скрытые нейроны и связи только тогда, когда это даёт выигрыш по приспособленности.

  Это фактически минимизирует размерность пространства поиска на всех этапах эволюции и делает обучение существенно более эффективным.
\end{enumerate}

Также подробно описаны два типа структурных мутаций:

\begin{itemize}

  \item добавление связи между двумя ранее не связанными узлами;
  \item расщепление существующей связи добавлением нового скрытого узла (при этом старая связь дизактивируется, а новые получают веса, минимизирующие резкое ухудшение функции).

\end{itemize}

\subsubsection{Связь с задачей моделирования когнитивного агента}

Результаты Stanley и Miikkulainen \cite{stanley2002neuroevolution} обосновывают выбор NEAT как базового нейроэволюционного алгоритма для моделирования когнитивного агента. NEAT:

\begin{itemize}

  \item естественно работает в задачах \textbf{обучения с подкреплением}, где поведение агента задаётся нейросетью
  % TODO уточнить
  \item хорошо справляется с задачами с частичной \textbf{наблюдаемостью и памятью} (за счёт рекуррентных связей и эволюции структуры)
  \item позволяет \textbf{избежать ручного подбора архитектуры}, что является серьёзной проблемой в классическом RL-подходе, где архитектура фиксируется заранее.

\end{itemize}

Это даёт чёткую исследовательскую гипотезу:
  
''Эволюция топологии нейросети когнитивного агента методом NEAT может обеспечить более высокую эффективность обучения (скорость, устойчивость, компактность модели) по сравнению с классическим RL-подходом при использовании фиксированной архитектуры сети.''
